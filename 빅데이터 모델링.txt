완전성

	데이터 중복으로 데이터 일관성을 유지하기 어려움 
비지니스 룸

	제공하는것, 모든사용자가 해당규칙에 대해 동일한 판든을하고 데이터를 조작하할수 있음
좋은 데이터 모델의 요소
데이터 재사용
	데이터의 재사용성을 향상하고싶으면 데이터의 통합성,독립성에대해 고려

안정성 및 확장성

관계형 모델 이론
DB의 구성3요소
데이터 구조:사용자가 데이터를 인식하는구성
	6가지 특성을 가진 행과 열로 구성된 2차원관계테이블로 표현
	1. 각 열은 하나의 값 (1차 정규화)
	2.각 열은 동일한 성격의 값을 가진다
 	3.각 행은 일련의 열의 값에의해 유일하게 식별된다 (Primary Key)
	4.열의 순서 의미 없음
	5.행의 순서 의미없음
	6.각 열은 유일한 이름을 가진다
데이터 조작 : 사용자가 관계테이블에 행하는 일련의 처이형태
	SET처리 가능(한번에)
	관계 연산자 : 조회
데이터 무결성	
	실체 무결성 규칙(entity imtegrity)
	=Primary key는 Not Null이며,하나 이상의 컬럼에 의해서 유일하게 식별 

	참조무결성(referential integrity rule)
	= 관계 데이터 의 모든 외부 식별자 값은 연관되어 있는 관계 테이블에 주식별자 값이 존재 	
		1.입력규칙
		부모와 자식테이블 중에서 자식테이블에 자료를 입력할 때 외부키 (foreign key)가
		되는 칼럼의 값이 부모 테이블에 존재유무를 따져 입력 유무를 따지는 규칠
		2.삭제규칙
		부모와 자식 테이블중에서 부모 테이블에 자료를 삭제할때 주키(primary key)로 선언된
		컬럼의 값이 자식테이블에사용되고있는지를 따져서 삭제할지정하는거
	영억(속성)무결성 규칙(domain integrity rule)
	=동일한 속성에 관한 무결성 규칙으로 데이터 타입,길이,허용 값,기본값,유일성,null 
	  여부등에 관한 제한 
데이터 모델 3요소
	업무가 다루는 사항 => 엔티티(개체,실체) entity

	엔티티 사이에 존재하는 연관 => 관계 relationship
	=항상 2테이블간에 있다
		식별성
		선택성
		기수성 
	엔티티가 가지고있는 상세한 특성 => 속성 attribute
